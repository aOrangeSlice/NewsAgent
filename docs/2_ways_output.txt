这个程序里其实有两种“简报输出路径”：

LLM 生成版
规则模板版（fallback，通常你说的“简报版本”）
核心分流逻辑在 llm.py:90：

如果 use_llm 是 false，直接走 fallback
如果 use_llm 是 true 且 Ollama 可用，走 LLM
如果 LLM 调用失败，也会回退到 fallback
你当前配置里 use_llm 已经是 true，见 settings.json:25。

具体输出差异如下。

一、结构差异（最直观）

规则模板版：
固定章节，基本恒定：
市场先看
各地区主流媒体重要新闻 Top 5
医疗与健康资讯 Top 5
AI 与科技
判断逻辑
来源
参考样例：briefing_33.md
LLM 版：
章节标题和顺序不固定，取决于模型生成
你当前 latest 样例是：
Market Data Overview
News Highlights
Key Takeaways
参考样例：latest.md


二、内容粒度差异

规则模板版：
每条新闻几乎都是“故事ID + 标题 + 简短摘要 + 来源URL”
强结构化，像数据库导出后的可读报告
末尾有大段来源清单（最多 40 条），可追溯性强
LLM 版：
更偏“分析叙述”，会做归纳和总结
通常不保留 story id
来源引用是否完整依赖模型是否遵守指令

三、可控性与稳定性

规则模板版：
稳定、可复现、几乎不漂移
同一批输入多次生成，格式基本一致
LLM 版：
可读性和“像分析师”程度更好
但存在输出漂移和不一致风险


四、程序里对两者的“硬规则”

LLM 版提示词在 llm.py:119：
要求“只用证据 JSON”
要求“每个事实 bullet 都带 source URL”
要求市场按三组（全球指数/美股板块/商品外汇）
要求医疗板块始终出现
LLM 输出后处理在 llm.py:310 与 llm.py:323：
清理 thinking 痕迹
删除“无相关证据”之类空段落
规则模板版实现在 llm.py:155：
世界新闻按地区最多 5 条
医疗最多 5 条
AI 科技最多 8 条
来源列表最多 40 条


五、你当前样例里体现出的关键现象

briefing_33.md 是典型规则模板输出：有固定中文章节、story id、大段来源列表。
latest.md 是典型 LLM 风格输出：英文分析体、章节自由、无 story id。
这说明你环境里两种输出都出现过，且 LLM 版可读性更强，但约束执行不一定 100% 严格。
