## 快速检查（12 项）

- [ ] **1. 程序状态正常**  
  `python -m newsagent doctor` 可以正常执行，数据库、Ollama 和邮件状态符合预期。
  当前基线示例：
  ```json
  {
    "sources": 42,
    "enabled_sources": 33,
    "llm_provider": "ollama",
    "llm_model": "qwen3:8b",
    "ollama_available": true,
    "email_enabled": true,
    "email_configured": true
  }
  ```

- [ ] **2. 新闻采集正常**  
  运行 `python -m newsagent collect --limit 3` 后，有新数据或明确显示“没有新数据”，且失败来源不会中断整个任务。
  访问所有启用的信息源；每个来源最多获取 3 条；保存新内容到 `data/newsagent.db`；对内容进行初步去重和聚类。
  输出类似结果：
  ```json
  {
    "inserted": 81,
    "clustered": 7,
    "errors": [
      {
        "source": "npj_digital_medicine",
        "error": "not well-formed (invalid token): line 14, column 4"
      }
    ]
  }
  ```

- [ ] **3. 简报能够生成**  
  `python -m newsagent daily --output-language zh --collect-limit 3 --brief-limit 20` 可以成功生成 Rules 和 LLM 两版文件。
  检查 `data/outbox/latest.md`、`data/outbox/latest_rules.md`、`data/outbox/latest_llm.md` 是否都被更新。

- [ ] **4. 没有乱码**  
  中文、英文、日文、URL 和数字均正常显示，没有明显乱码或损坏字符。

- [ ] **5. 时间与行情正确**  
  新闻日期在合理时间范围内；行情注明实时、盘前、盘后或上一交易日收盘，不把旧价格当成当前价格。

- [ ] **6. 内容有可靠来源**  
  重要新闻、数字和结论都有可打开的来源链接；抽查至少 3 条，内容与来源基本一致。

- [ ] **7. 栏目基本完整**  
  简报至少包含市场、国际/地区新闻、医疗健康、AI/科技；无数据时有明确说明。

- [ ] **8. 没有明显重复**  
  同一 URL、同一事件、同一股票或指数没有重复出现。

- [ ] **9. 重点内容合理**  
  最重要的事件排在前面，没有被娱乐性、过旧或低价值内容挤占。

- [ ] **10. 摘要简洁且没有乱编**  
  摘要能说明“发生了什么、为什么值得关注”，没有来源中不存在的原因、预测或结论。

- [ ] **11. Rules 与 LLM 比较**  
  两版基于大致相同的新闻；记录哪一版更完整、准确、易读。若 LLM 明显遗漏或乱编，选择 Rules 版。

- [ ] **12. 投递与安全正常**  
  测试邮件能正常显示中文且不会重复发送；密码和 Token 没有明文写入配置、日志或 Git。
