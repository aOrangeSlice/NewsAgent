新闻采集
  ↓
原文标题、原文摘要直接入库
  ↓
聚类、评分、选取新闻
  ↓
同时生成两份原文简报
  ├─ 规则版 rules
  └─ LLM 版 llm
  ↓
根据 --output-language 决定是否翻译
  ↓
保存最终简报并输出/发送


--------------------------------------------
完整原文简报
→ 按 Markdown 段落切成约 4,500 字符一块
→ 暂时替换 URL 和 [新闻编号]
→ 分块调用 LLM 翻译
→ 恢复 URL 和编号
→ 合并全部翻译块
→ 检查有没有链接或编号丢失

数据库中的简报保存
每份简报会同时保存：
canonical_body：翻译前的原文简报
body：最终输出语言版本
generation_mode：rules 或 llm
generation_model：LLM 版使用的模型
translation_status：翻译是否成功
translation_model：翻译使用的模型
briefing_group：标记两份简报属于同一轮

出库阶段
daily 会生成：
latest_rules.md   # 规则版
latest_llm.md     # LLM 版
latest.md         # 当前默认版本

config/settings.json 中：
"use_llm": false
表示：
latest.md 使用规则版。
邮件发送规则版。
LLM 版仍然会生成和保存。

改成 true 后，latest.md 和邮件使用 LLM 版，但规则版仍会保存。

所以可以概括为：新闻入库保持原文不变；修改的是简报出库前增加“双版本生成、末端分块翻译、校验和回退”。
